# Proposta por e-mail

Objetivo: fazer o cliente ABRIR o link da página nova.

## Regras

1. **Sem preço.** Preço no primeiro e-mail vira spam mental.
2. **Rapport real primeiro.** Elogio específico: nota Google, avaliações, credencial real.
3. **Defeito com delicadeza.** 1-2 pontos objetivos do site atual como oportunidade.
4. **A entrega é o argumento.** "Preparei uma nova versão" + link publicado.
5. **CTA leve.** Pedir só que abra e diga o que achou.
6. **Curto.** 120-180 palavras.

## Estrutura

- **Assunto**: específico. Ex.: `Uma nova versão do site da [Nome] — [nota] no Google merece mais`
- **P1**: quem encontrou + elogio específico
- **P2**: observação sobre site atual
- **P3**: "nova versão, já no ar" + link novo
- **P4**: CTA — abrir no celular, responder com impressão
- **Assinatura**: nome, apresentação e WhatsApp do config

## Envio (Cursor)

- Modo **rascunho** (padrão): abra Gmail via MCP `cursor-ide-browser`, crie rascunho com destinatário, assunto e corpo. Avisar para anexar `preview.png` antes de enviar.
- Modo **enviar direto**: Gmail web via browser, ou MCP Resend se configurado.
- Sem e-mail confirmado: sugerir WhatsApp com mensagem adaptada.

## Depois do envio

Registrar em `leads.md` (status `proposta enviada` + data). Follow-up educado após 3-4 dias úteis sem resposta.
